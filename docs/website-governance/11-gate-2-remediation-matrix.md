## PRIORITY DEFINITIONS

### P0 — Critical (Blocks Launch)
- Unsupported capability claims (false features)
- Placeholder pages with no content
- Grounding <90% for core pages (Homepage, Platform, Solutions, Security, About)
- Copied competitor language

### P1 — High (Launch Blocker if Not Fixed)
- Grounding <85% for role/industry pages
- Generic SaaS template language
- Weak commercial positioning
- Technical overflow on buyer pages

### P2 — Medium (Post-Launch Enhancement)
- Minor grounding gaps (1-5% below threshold)
- Optional content enhancements
- Example content validation
